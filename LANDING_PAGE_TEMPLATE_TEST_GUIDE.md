# 랜딩페이지 템플릿 적용 테스트 가이드

## 🎯 테스트 목적
템플릿이 랜딩페이지 생성 시 정확히 적용되는지 **실제로** 확인

## 📋 테스트 단계

### 1단계: 템플릿 확인
```
URL: https://superplace-academy.pages.dev/dashboard/admin/landing-pages/templates
```
- 템플릿 목록 확인
- 기본 템플릿(⭐ 표시) 확인
- HTML 내용 확인 (템플릿 클릭 → HTML 보기)

### 2단계: 랜딩페이지 생성
```
URL: https://superplace-academy.pages.dev/dashboard/admin/landing-pages/create
```

**입력 정보:**
1. **학생 선택**: 아무 학생이나 선택
2. **제목**: "템플릿 테스트"
3. **부제목**: "템플릿 적용 확인"
4. **기간**: 2024-01-01 ~ 2024-03-31
5. **템플릿**: 기본 템플릿 자동 선택 (변경 안 함)
6. **데이터 옵션**: 모두 체크된 상태 유지

**생성 버튼 클릭 ✅**

### 3단계: 생성 결과 확인
생성 성공 시 알림창에 URL 표시:
```
https://superplace-academy.pages.dev/lp/lp_XXXXXXXXX
```

### 4단계: 실제 페이지 접속 및 확인

**✅ 정상인 경우:**
- 템플릿 디자인이 적용됨
- 학생 이름, 학원명 등 변수가 치환됨
- CSS 스타일이 적용됨
- 반응형 디자인 작동

**❌ 문제가 있는 경우:**
- 빈 페이지
- "페이지를 찾을 수 없습니다" 오류
- HTML 코드가 그대로 보임
- 변수 ({{studentName}} 등)가 치환되지 않음

## 🔍 문제 발생 시 확인 사항

### A. 브라우저 콘솔 (F12) 확인
- 네트워크 탭 → 페이지 응답 확인
- 콘솔 탭 → JavaScript 오류 확인

### B. 페이지 소스 보기 (Ctrl+U)
- HTML이 있는지 확인
- 템플릿 구조가 있는지 확인
- 변수가 치환되었는지 확인

### C. 랜딩페이지 목록 확인
```
URL: https://superplace-academy.pages.dev/dashboard/admin/landing-pages
```
- 생성된 페이지가 목록에 나타나는지 확인
- 상태가 "활성"인지 확인

## 📊 테스트 결과 보고

다음 정보를 제공해 주세요:

1. **템플릿 선택 시**: 템플릿이 선택되었는지? (이름 표시됨?)
2. **생성 성공 여부**: 알림창에 URL이 표시되었는지?
3. **페이지 접속 결과**: 
   - ✅ 템플릿이 적용되어 정상 표시
   - ❌ 문제 발생 (구체적으로 무엇이 보이는지)
4. **스크린샷**: 가능하면 실제 페이지 화면 캡처

## ⚡ 빠른 테스트 (간단 버전)

1. `/dashboard/admin/landing-pages/create` 접속
2. 학생 선택 + 제목 "테스트" 입력 + 기간 선택
3. **생성** 버튼 클릭
4. 알림창의 URL로 접속
5. 페이지가 제대로 보이는지 확인
6. 결과 보고
