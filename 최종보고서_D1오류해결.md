## 🔍 문제 확인 결과

### 실제 테스트 결과
```
POST https://superplacestudy.pages.dev/api/bot-purchase-requests/create
응답: 500 Internal Server Error
오류: D1_ERROR: table BotPurchaseRequest has no column named email: SQLITE_ERROR
```

### 원인 분석
- SQLite의 `CREATE TABLE IF NOT EXISTS` 구문은 기존 테이블이 있으면 새 컬럼을 추가하지 않습니다
- BotPurchaseRequest 테이블은 이미 존재하지만 email, name, requestAcademyName, phoneNumber 컬럼이 없는 상태입니다
- 따라서 **수동으로 컬럼을 추가해야 합니다**

## ⚠️ 필수 실행 작업

### 🎯 Cloudflare D1 콘솔에서 마이그레이션 실행 (약 3분)

**1단계: Cloudflare Dashboard 접속**
```
https://dash.cloudflare.com
```

**2단계: 프로젝트 찾기**
- Workers & Pages 메뉴 클릭
- **superplacestudy** 프로젝트 선택

**3단계: D1 콘솔 열기**
- **Settings** 탭 클릭
- **Bindings** 섹션 스크롤
- **D1 Database** 찾기
- **"Open D1 console"** 버튼 클릭

**4단계: SQL 실행 (다음 4개 명령어를 하나씩 실행)**
```sql
ALTER TABLE BotPurchaseRequest ADD COLUMN email TEXT;
```
```sql
ALTER TABLE BotPurchaseRequest ADD COLUMN name TEXT;
```
```sql
ALTER TABLE BotPurchaseRequest ADD COLUMN requestAcademyName TEXT;
```
```sql
ALTER TABLE BotPurchaseRequest ADD COLUMN phoneNumber TEXT;
```

💡 **주의**: `duplicate column name` 오류가 나오면 해당 컬럼은 이미 존재하는 것이므로 무시하고 다음 명령어를 실행하세요.

**5단계: 검증**
```sql
PRAGMA table_info(BotPurchaseRequest);
```

다음 컬럼들이 표시되어야 합니다:
- ✅ `email`
- ✅ `name`
- ✅ `requestAcademyName`
- ✅ `phoneNumber`

## 💡 Troubleshooting

### Q: 마이그레이션 후에도 같은 오류가 발생해요
**A**: 다음을 확인하세요:
1. Cloudflare Pages 배포 완료 대기 (1-2분)
2. 브라우저 캐시 클리어 (Ctrl+Shift+R 또는 Cmd+Shift+R)
3. D1 콘솔에서 다시 `PRAGMA table_info(BotPurchaseRequest);` 실행
4. email, name, requestAcademyName, phoneNumber 컬럼이 있는지 확인

### Q: D1 콘솔을 찾을 수 없어요
**A**: 
1. https://dash.cloudflare.com 접속
2. 왼쪽 메뉴에서 "Workers & Pages" 클릭
3. "superplacestudy" 프로젝트 클릭
4. 상단 "Settings" 탭 클릭
5. 아래로 스크롤하여 "Bindings" 섹션 찾기
6. "D1 Database" 항목에서 "Open D1 console" 버튼 클릭

### Q: "duplicate column name" 오류가 나와요
**A**: 정상입니다! 해당 컬럼은 이미 존재하므로 다음 ALTER TABLE 명령어를 계속 실행하세요.
