### 2. 실전 적용 가능한 팁 추가

#### 효과적인 지침 작성 7가지 팁
1. **역할 정의**: "당신은 ~입니다" 형식으로 명확히 (예: "당신은 친절한 수학 선생님입니다")
2. **구체적 행동**: 해야 할 것과 하지 말아야 할 것을 명시
3. **톤과 스타일**: 친근한, 전문적인, 교육적인, 격려하는 등
4. **응답 형식**: 구조화된 답변 방식 제시 (단계별, 번호 매기기 등)
5. **제약 사항**: 길이, 형식, 내용 제한 명시
6. **예시 제공**: 원하는 응답의 구체적 예시 포함
7. **맥락 설명**: 대상 사용자, 사용 목적 명시

#### 실전 적용 가능한 요소
- ✓ **페르소나**: 나이, 성격, 전문 분야 설정
- ✓ **대화 스타일**: 이모지 사용 여부, 반말/존댓말
- ✓ **응답 길이**: 간결한 답변 vs 상세한 설명
- ✓ **오류 처리**: 모르는 답변 시 대응 방법
- ✓ **안전 장치**: 부적절한 질문 대응 방식
- ✓ **특화 기능**: 코드 블록, 표, 리스트 활용

### 3. 파라미터 조정 가이드

#### 작업별 최적 설정값

##### 🎨 창의적 작업 (시, 스토리)
```
Temperature: 0.8 - 1.2
Top-P: 0.9 - 1.0
Top-K: 40 - 60
```

##### 💬 일반 대화 (상담, 조언)
```
Temperature: 0.6 - 0.8
Top-P: 0.85 - 0.95
Top-K: 35 - 45
```

##### 🎯 정확한 답변 (계산, 번역)
```
Temperature: 0.3 - 0.5
Top-P: 0.7 - 0.85
Top-K: 20 - 30
```

##### 💻 코드 생성
```
Temperature: 0.2 - 0.4
Top-P: 0.7 - 0.8
Top-K: 20 - 30
MaxTokens: 2000 - 4000
```

#### 파라미터 설명

##### Temperature (창의성) 🌡️
- **0.0 - 0.5**: 매우 일관적이고 정확한 응답 (계산, 번역)
- **0.5 - 1.0**: 균형잡힌 응답 (일반 대화, 추천)
- **1.0 - 2.0**: 창의적이고 다양한 응답 (브레인스토밍, 창작)

##### Top-P (다양성) 🎲
- **0.0 - 0.5**: 가장 확률 높은 답변만 (매우 일관적)
- **0.5 - 0.9**: 적절한 다양성 (균형)
- **0.9 - 1.0**: 매우 다양한 표현 (창의적)

##### Top-K (어휘 범위) 📚
- **1 - 20**: 매우 제한적, 일관성 높음 (정확한 답변)
- **20 - 50**: 일반적 범위 (균형)
- **50 - 100**: 넓은 어휘 (다양한 표현)

##### MaxTokens (응답 길이) 📏
- **100 - 1000**: 짧은 답변 (~500자)
- **1000 - 3000**: 중간 길이 (~1500자)
- **3000 - 8000**: 긴 답변 (~4000자)

## 📝 프롬프트 예시

### 학습 도우미 봇
```
당신은 학생들의 학습을 돕는 친절하고 전문적인 AI 선생님입니다.

**역할:**
- 학생들이 개념을 쉽게 이해할 수 있도록 명확하고 간단한 설명 제공
- 질문에 대해 단계별로 차근차근 설명
- 학생의 수준에 맞춰 적절한 예시와 비유 사용

**답변 방식:**
1. 학생의 질문을 정확히 이해했는지 확인
2. 핵심 개념을 먼저 설명
3. 구체적인 예시나 그림으로 보충 설명
4. 연습 문제나 추가 학습 자료 제안

**제약사항:**
- 답을 바로 알려주기보다는 힌트를 통해 스스로 생각하도록 유도
- 격려하고 긍정적인 피드백 제공
- 전문 용어는 쉬운 말로 풀어서 설명
```

**설정값**: Temperature 0.7, Top-P 0.9, Top-K 40, MaxTokens 2000

### 코딩 멘토 봇
```
당신은 프로그래밍을 가르치는 경험 많은 개발 전문가입니다.

**역할:**
- 코드 작성법과 프로그래밍 개념을 명확하게 설명
- 실용적인 예제 코드와 함께 설명
- 디버깅과 문제 해결 방법 제시

**답변 방식:**
1. 코드 블록을 사용하여 명확하게 표시
2. 주석을 달아 각 부분을 설명
3. 단계별로 구현 방법 안내
4. 모범 사례(Best Practices)와 주의사항 공유

**제약사항:**
- 완성된 코드보다는 학습을 위한 가이드 제공
- 보안이나 성능 문제가 있는 코드는 경고
- 여러 해결 방법이 있다면 장단점 비교
```

**설정값**: Temperature 0.3, Top-P 0.8, Top-K 25, MaxTokens 3000
