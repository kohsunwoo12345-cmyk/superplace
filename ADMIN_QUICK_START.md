## 🔧 일반 사용자를 관리자로 만들기

### 방법 1: 명령어 사용 (서버 접근 필요)

```bash
# 1. 프로젝트 폴더로 이동
cd /home/user/webapp

# 2. 사용자를 관리자로 승격
node make-admin.js <이메일주소>

# 예시
node make-admin.js newadmin@company.com

# 3. 결과 확인
✅ newadmin@company.com 사용자가 SUPERADMIN으로 승격되었습니다!
```

### 방법 2: 현재 등록된 사용자 확인

```bash
# 모든 사용자 목록 보기
node check-users.js

# 출력 예시:
=== 등록된 사용자 ===
[
  {
    "email": "test@example.com",
    "name": "테스트 사용자",
    "role": "SUPERADMIN",     ← 관리자
    "createdAt": "2025-10-30"
  },
  {
    "email": "user2@example.com",
    "name": "두번째 사용자",
    "role": "USER",            ← 일반 사용자
    "createdAt": "2025-10-30"
  }
]
```

- ✅ 전체 등록 사용자 수

- ✅ 구독률 (%)
- ✅ 플랜별 구독자 분포

- ✅ 이번 달 총 매출
- ✅ 전월 대비 성장률
- ✅ 결제 내역

### 4. 플랫폼 연동
- ✅ 전체 연동 가능한 플랫폼 수
- ✅ 실제 연결된 계정 수

## 🎬 실제 사용 시나리오

### 시나리오 A: 새 직원을 관리자로 추가

```
1️⃣ 새 직원이 회원가입
   → 이메일: newstaff@company.com
   → 자동으로 "USER" 권한 부여

2️⃣ 서버에서 관리자로 승격
   터미널 실행:
   $ node make-admin.js newstaff@company.com
   ✅ 승격 완료!

3️⃣ 직원이 로그아웃 후 재로그인
   → 좌측 사이드바에 "관리자" 메뉴 나타남

4️⃣ 관리자 대시보드 접근 가능
   → 모든 통계 확인 가능
```

### 시나리오 B: 월말 매출 확인

```
1️⃣ 관리자 계정으로 로그인

2️⃣ "관리자" 메뉴 클릭

3️⃣ 대시보드에서 확인:
   📊 이번 달 매출: ₩xxx,xxx
   📈 전월 대비: +x%
   👥 신규 가입자: x명
   💳 활성 구독: x개
```

### 시나리오 C: 사용자 현황 파악

```
1️⃣ 관리자 대시보드 접속

2️⃣ "사용자 관리" 섹션에서:
   - 최근 가입한 사용자 10명 확인
   - 각 사용자의 권한 확인
   - 이메일 및 가입일 확인

3️⃣ "구독 현황" 섹션에서:
   - 플랜별 가입자 수 확인
   - 무료 사용자 vs 유료 사용자 비율 파악
```

## ❓ 자주 묻는 질문 (FAQ)

### Q1: 관리자 메뉴가 안 보여요!
**A:** 다음을 확인하세요:
1. 계정이 ADMIN 또는 SUPERADMIN 권한인지 확인
2. 로그아웃 후 다시 로그인
3. 브라우저 새로고침 (Ctrl + F5)

### Q2: 권한을 변경했는데 적용이 안 돼요
**A:** 세션을 초기화해야 합니다:
1. 로그아웃
2. 브라우저 완전히 닫기
3. 다시 로그인

### Q3: 관리자 권한을 다시 일반 사용자로 되돌리고 싶어요
**A:** 데이터베이스를 직접 수정해야 합니다:
```sql
UPDATE User SET role = 'USER' WHERE email = 'user@example.com';
```

### Q4: 여러 명을 한 번에 관리자로 만들 수 있나요?
**A:** 각각 명령어를 실행해야 합니다:
```bash
node make-admin.js admin1@example.com
node make-admin.js admin2@example.com
node make-admin.js admin3@example.com
```
